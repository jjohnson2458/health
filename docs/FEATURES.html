Core Features

Dashboard
GET /dashboard

The main landing page after login. Displays:

  • Today's entry status - quick link to add or edit today's data
  • Current streak - consecutive days of tracking
  • Latest weight - most recent recorded weight
  • Active medications - count with link to manage
  • Upcoming appointments - next scheduled visits
  • 7-day trend charts - weight, calories, exercise at a glance
Auth required
Health Entries
GET /entry GET /entries

Daily health data logging with one entry per day. Fields tracked:

  • Weight (lbs) - encrypted at rest
  • Calories - daily intake
  • Protein (g), Carbs (g), Fat (g)
  • Heart Rate (bpm)
  • Blood Sugar (mg/dL)
  • Exercise - type + minutes
  • Notes - encrypted at rest

Unique constraint: one entry per user per date. Edit and delete supported.

Auth required Encrypted
Analytics
GET /analytics GET /analytics/data

Interactive Chart.js visualizations with date range filtering:

  • Weight Over Time - line chart tracking progress
  • Calories Over Time - daily intake patterns
  • Heart Rate & Blood Sugar - vitals monitoring
  • Exercise Minutes - activity trends
  • Day-of-Week Averages - identify patterns (Mon-Sun)

Date ranges: Week, Month, 3 Months, 6 Months, Year. Data fetched via AJAX.

Auth required

Health Tools

Calorie Deficit Calculator
GET /calculator/calories

Uses the Mifflin-St Jeor equation to calculate:

  • BMR (Basal Metabolic Rate) - calories burned at rest
  • TDEE (Total Daily Energy Expenditure) - adjusted for activity level
  • Recommended daily calories based on goal (lose, maintain, gain)

Inputs: age, gender, height, weight, activity level (sedentary to very active).

Macro Calculator
GET /calculator/macros

Calculates ideal macronutrient split based on daily calorie target and goal:

  • Protein - grams and calories (4 cal/g)
  • Carbohydrates - grams and calories (4 cal/g)
  • Fat - grams and calories (9 cal/g)

Goal-based splits: weight loss (high protein), maintenance (balanced), muscle gain.

Food Tracker
GET /calculator/food

Noom-style food categorization system with 120+ foods:

Green Low-calorie density

Fruits, vegetables, whole grains

Yellow Moderate density

Lean meats, dairy, beans

Red High-calorie density

Oils, sweets, processed foods

Includes custom food entry form for items not in the database.

Weight Loss Planner
GET /planner GET /planner/data

Goal-setting tool with milestone projections:

  • Set current weight, goal weight, and weekly target (0.5-2 lbs/week)
  • Calculates weeks to goal with projected milestone dates
  • Chart comparing projected vs actual weight over time
  • Personalized recommendations (lifestyle, supplements, medication considerations)
Always consult your healthcare provider before starting any weight loss program.

Medical Management

Medications
GET /medications

Full prescription tracking with HIPAA-compliant audit trail:

  • Add/Edit medications - name, dosage, frequency, prescriber, prescribed date
  • Status management - active, inactive, discontinued (with reason)
  • Reactivation - restore discontinued medications
  • History log - complete audit trail of all changes per medication
  • Source tracking - manual entry, provider push, or CSV import
  • Share view - printable medication list for providers
Encrypted HIPAA Audited
Appointments
GET /appointments GET /appointments/calendar

Medical appointment scheduling and tracking:

  • Types: checkup, lab, specialist, dental, vision, therapy, other
  • Calendar view - monthly calendar with appointment markers
  • Status flow: scheduled → completed / cancelled
  • Reminder system - email and SMS flags for next-day appointments
  • Provider & location - encrypted at rest
Encrypted

Security & Compliance

Authentication
  • Registration - email verification required
  • Login - email + password
  • Two-Factor Auth - 6-digit email code (10-min expiry)
  • Forgot Password - token-based email reset
  • Rate Limiting - 5 attempts / 15 min per email+IP
  • Session Timeout - 30 min inactivity
  • CSRF Protection - all POST/PUT/DELETE forms
  • Bcrypt - cost factor 12
Encryption at Rest

AES-256-CBC encryption for all personally identifiable information:

User PII

  • First name, Last name
  • Email address
  • Phone number

Health Data

  • Weight
  • Notes

Medical Data

  • Medication names, dosage
  • Prescriber info
  • Provider names, locations

Email lookups use SHA-256 hash (email_hash) for fast queries without decryption.

HIPAA Compliance
  • Encryption at rest - AES-256-CBC for all PII and PHI
  • Encryption in transit - SSL/TLS required in production
  • Access controls - role-based (user/admin), session authentication
  • Audit trail - all data access logged with user ID, action, IP, timestamp
  • Minimum necessary - users can only access their own data
  • Security headers - CSP, X-Frame-Options, X-Content-Type-Options, X-XSS-Protection
  • Error handling - sensitive data never exposed in error messages (production mode)
Audit Logging

Every significant action is recorded in the audit_log table:

  • Actions tracked: login, view, edit, delete, export, admin operations
  • Fields logged: user_id, action, resource, details, ip_address, timestamp
  • Retention: 1 year (nightly cleanup purges older records)

Administration

Admin Dashboard
GET /admin

System overview with key metrics:

  • Total users, health entries, medications, appointments
  • Errors in last 24 hours (with link to error log)
Admin only
User Management (CRUD)
GET /admin/users

Full user lifecycle management:

  • Create - add new users with name, email, password, role, language
  • Read - paginated user list with decrypted PII, role badges, verification status
  • Update - change role (user/admin), email verification, language, password reset
  • Delete - cascade delete all user data (with confirmation)
  • Export - download individual patient data as CSV
Admin only Audit Logged
Error Log
GET /admin/errors

Paginated error log viewer showing application errors and exceptions. Captured via global error/exception handlers in public/index.php.

Fields: level, message, file, line, stack trace, user ID, IP, URL, timestamp.

Admin only
