About This Documentation

This documentation covers the VQ Healthy application's PHP backend, organized by namespace. The codebase follows an MVC architecture with a custom micro-framework.

  • Core — Framework foundation: Router, Database (PDO singleton), Session, View engine, Encryption (AES-256-CBC), Middleware base, Mailer, and abstract Model/Controller
  • App\Controllers — HTTP request handlers for authentication, dashboard, entries, medications, appointments, analytics, calculators, and admin
  • App\Models — Data access layer with field-level encryption for HIPAA-protected health information (PHI)
  • App\Middleware — Request filters for authentication, CSRF protection, guest-only routes, and admin authorization

HIPAA compliant: All PHI fields are encrypted at rest using AES-256-CBC. Audit logging tracks every data access.
